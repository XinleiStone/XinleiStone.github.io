VS Code助你Coding效率翻倍

Posted on 2019-01-15 | In 效率

简介

Visual Studio Code（简称 VS Code）是一个由微软开发的（TS + Electron），同时支持 Windows、Linux 和 macOS 系统且开放源代码的代码编辑器，它支持测试，并内置了 Git 版本控制功能，同时也具有开发环境功能，例如代码补全（类似于 IntelliSense）、代码片段和代码重构等，该编辑器支持用户个人化配置，例如改变主题颜色、键盘快捷方式等各种属性和参数，还在编辑器中内置了扩展程序管理的功能。

  • 我为什么放弃使用 Sublime——轻的不像 IDE，looks 不专业。主要还是因为插件较少、配置繁琐，不能可视化 settings。

  • 我为什么放弃使用 Atom——卡。个人非常喜欢 Atom 的界面风格，也已经习惯了它的快捷键操作，但是插件还没用热乎，写 banma_page 如果使用一些插件（比如 path 自动补全等）会卡爆。

  • 我为什么放弃使用 Webstorm——以前写 Java 就用的 idea，后来写前端用了 webstorm，真是好用。但是总是感觉不像写前端，总像是在写后端。需要购买（注册）。本来使用 VSCode 是为了补充打开文件或小项目的工具，没想到用了之后一发不可收拾。

也许 VSCode 集成了以上几个编辑器（IDE）的优点，比 sublime 开源，比 atom 更快，比 webstorm 更轻。

好在哪里？

  • 集成 Git——微软都收购 Git 了，你懂的

  • 轻量级、启动速度快，执行简单，调试方便

  • 智能感知——浓缩版的 VS（甩 sublime 和 atom 几条街）

  • 易调试

  • 可扩展 支持 javascript，C++，python，C#…

  • 跨平台、 Windows、Mac 和 Linux 通用

3、快捷键

建议大家把功能键触发改一下： 系统偏好设置 -> 键盘

  • cmd + T 快速切换某个文件 （不需要输入全拼或者特殊字符！）

  • cmd + shift + P / F1 万能键

  • cmd + shift + F 全文搜索

  • cmd + shift + H 全文替换

  • cmd + shift + K 删除行

  • cmd + shift + D 复制行

  • cmd + D 多选 （cmd + F2 全选）

  • ctrl + G 跳转到指定行

  • ctrl + tab 快速切换到上一个文件，多次按(shift)tab 可以来回选择，也可以直接点击文件顶部名称来显示历史文件列表进行切换

  • alt + ↑/↓ 调整代码行整体上下移动

  • alt + 点击多处可以多光标操作，也可以 alt+cmd+↑/↓ 多光标选择，但是不支持直接拖，没 sublime 方便

5、用户代码片段

打开方式——Mac 下：Code -> 首选项 -> 用户片段

  • prefix :代码片段名字，即输入此名字就可以调用代码片段。

  • body :这个是代码段的主体。需要编写的代码放在这里。

  • $1 :生成代码后光标的初始位置。

  • $2 :生成代码后光标的第二个位置，按tab键可进行快速切换，还可以有$3, $4, $5…..

  • ${1, 字符} :生成代码后光标的初始位置（其中 1 表示光标开始的序号，字符表示生成代码后光标会直接选中字符。）

  • description :代码段描述，输入名字后编辑器显示的提示信息。
